Outcome C
⚡ Fan out — N agents in ONE message
Each independent task runs concurrently. Critical: dispatch in a single message — separate messages run sequentially. Save N × wall-clock cost.
Outcome D
👑 Dispatch via apex
For multi-domain missions: route through hft-cto (or your project's apex), which orchestrates chiefs → leads → specialists with the consult protocol. Apex merges outputs + applies final clearance.
Rules
of thumb
Built-ins also exist — Plan (designs implementation plans without executing), general-purpose (full toolset, when no specialist fits). Use them as fallbacks, not first choices.  ·  If specialists disagree, call the architecture-review-board arbiter — it emits a binding ADR.  ·  Never spawn an agent that hallucinates a domain you don't have a specialist for. If the gap is real, build the specialist first.
