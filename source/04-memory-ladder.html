Memory ladder — hot, warm, cold

Three tiers of memory. Each loads at a different cost. Files only — no DB, no embeddings, no vendor lock-in.
🔥 HOT — every session
⚡ WARM — on relevance
🧊 COLD — forever archive
🔥 HOT
🔥Always loaded — every session start
~30K tokens fixed cost · drop with /clear or by trimming CLAUDE.md
~/.claude/CLAUDE.md
Global rules — Karpathy, MCP server docs, mission-workbook convention.
<repo>/CLAUDE.md
Project rules — committed, shared with team. Build commands, conventions.
<repo>/CLAUDE.local.md
Personal layer — gitignored, machine-specific paths, your own tweaks.
projects/<slug>/memory/MEMORY.md
Auto-memory index — one line per topical file (feedback / project / user / reference).
⚡ WARM
⚡Loaded on relevance — pulled by description match
Episodic + topical · indexed by MEMORY.md but loaded only when needed
repo/.claude/workbooks/
YYYY-MM-DD-*.md
Mission workbooks — episodic memory across multi-turn missions. Every dispatched agent reads + appends a handover block.
projects/<slug>/
memory/*.md
Topical memory files — feedback_*, project_*, user_*, reference_*. Pulled when the index entry matches.
~/.claude/plans/*.md
Plan-mode artifacts — useful when resuming a multi-day mission tomorrow morning.
🧊 COLD
🧊Cross-session archive — written by the Stop hook
~5–50 MB / week · grep-able forever · zero network, zero DB
~/claude-archive/
YYYY/MM/<project>/<session>/
Per-session snapshot: transcript.jsonl · MANIFEST.json · CLAUDE.md · workbooks · plans. Phase-1 lossless. Built atop these: an HNSW index when grep stops being fast enough.
~/claude-archive/.archive.log
Audit trail — one line per session archived: timestamp, session_id, project_slug, destination path. Tail it to confirm the Stop hook fired.
On every /exit the Stop hook snapshots HOT + WARM tiers into COLD — the only direction memory ever flows.
